# DeepSeek Engram：条件记忆机制如何重塑大语言模型的稀疏性维度

> **摘要**：2026年1月，DeepSeek联合北京大学发布了Engram论文，首次提出"条件记忆"作为大语言模型稀疏性的新维度。本文深入解析这一技术的核心原理、性能优势及其对下一代模型架构的变革性影响。

## 一、问题：大模型的"无效内耗"

理解Engram的价值，首先要认清当前大模型的核心缺陷。

语言建模本质上需要完成**两种截然不同**的任务：

| 任务类型 | 示例 | 本质 |
|---------|------|------|
| **组合式推理** | 分析文章逻辑结构、撰写商业计划书、解数学题 | 动态思考、灵活组合知识 |
| **知识检索** | 识别"亚历山大大帝"、判断"四大发明"这类实体 | 高频重复的静态模式，查表即可 |

然而，现有Transformer架构**没有专门的"档案库"**来处理这些静态模式。不管是复杂推理还是简单实体识别，都要动用核心计算资源——多层前馈网络和注意力机制——重新推演一遍。

这就好比让资深策划师每次写方案前，都要先核对公司名称的正确写法和整理基础行业术语。本该用于构思创意的精力，全耗在了重复性的基础工作上。

### MoE的局限：条件计算解决不了知识存储问题

MoE（Mixture-of-Experts）通过**条件计算**实现了稀疏激活——根据输入动态选择专家模块，减少无效计算。但MoE本质上仍是"动态计算"，它解决的是*如何计算*的问题，而非*知识存储*的问题。

> **核心矛盾**：Transformer缺乏原生的知识查找原语，只能通过计算来模拟检索，造成严重的效率内耗。

## 二、Engram核心思想：条件记忆与查算分离

DeepSeek提出的**条件记忆（Conditional Memory）**是一种互补的稀疏性维度，核心思想是：

> **把"记忆"与"计算"解耦——让静态知识的存储和检索交给专门的记忆模块，让Transformer专注于动态推理。**

Engram模块就是这个思想的具体落地实现——一个高效、可扩展的AI"档案库"。

### 与MoE的对比

| 维度 | MoE（条件计算） | Engram（条件记忆） |
|------|----------------|-------------------|
| **激活方式** | 动态路由专家 | 确定 性查找嵌入表 |
| **时间复杂度** | O(1) 但需计算 | **O(1) 纯查找** |
| **存储位置** | GPU HBM | 可卸载至CPU/SSD |
| **负责内容** | 如何处理 | 记住什么 |
| **稀疏性来源** | 专家激活 | N-gram模式匹配 |

## 三、技术原理：Engram如何工作

Engram不是替换Transformer，而是作为**新增模块**融入其中。核心工作流程只有两步：**检索**（从档案库找资料）和**融合**（把资料精准用到当前任务）。

### 3.1 检索阶段：构建"智能档案库"

#### 1. 分词器压缩

普通分词器为了保证文本完整还原，会给语义相同的词分配不同ID（如"Apple"和"apple"、"北京"和"北京市"），造成存储冗余。

Engram通过预定义映射函数，将语义等价的词统一成标准化ID。实验显示，这种方法让128k词汇量的分词器**瘦身23%**。

#### 2. 多头哈希（Multi-head Hashing）

要给所有可能的N-gram组合（如"亚历山大大帝"、"四大发明之一"）单独建档案，规模会指数级爆炸。

Engram采用**多头哈希**策略：
- 给每个N-gram分配多个独立的"哈希头"
- 通过确定性哈希函数，将短语快速映射到档案索引
- 多个哈希头的结果整合成完整的记忆向量

这种设计既避免了"找不到"的情况，又不会重复存储。

### 3.2 融合阶段：上下文感知的精准筛选

检索到的资料是通用的，不一定适合当前任务。融合阶段的核心是**上下文感知门控机制**：

```
门控值 = Similarity(当前上下文隐藏状态, 记忆向量)
```

- 相似度越高 → 门控值接近1 → 资料被采纳
- 相似度越低 → 门控值接近0 → 资料被过滤

之后通过轻量级卷积操作增强适用性，再通过残差连接融入模型计算流。

### 3.3 与mHC架构的集成

Engram支持与mHC（流形约束超连接）架构高效配合，采用"共享+独立"的参数策略：

| 参数类型 | 策略 | 作用 |
|---------|------|------|
| 稀疏嵌入表 | 所有分支共享 | 避免资料重复存储 |
| 值投影矩阵 | 所有分支共享 | 统一整理标准 |
| 键投影矩阵 | 每个分支独立 | 个性化筛选标准 |

## 四、突破GPU内存墙：系统级优化

Engram的价值不仅在于分工提升效率，更在于解决了GPU内存限制这一行业痛点。

### 4.1 确定性寻址带来的预取优化

传统MoE的路由决策依赖运行时隐藏状态，**无法预知**需要访问哪些参数，只能把所有参数堆在高速HBM里。

Engram的检索索引**只依赖词元序列**，在计算前就能精准预知内存地址，实现：

- **预取-重叠策略**：模型处理当前层时，异步预取下一层需要的嵌入向量
- **实验数据**：在NVIDIA H800上，附加100B参数的Engram嵌入表（存于CPU内存），4B模型吞吐量惩罚仅**1.9%**，8B模型仅**2.8%**

### 4.2 多级缓存层次结构

自然语言的N-gram遵循**Zipf分布**——少数高频模式占据绝大多数访问请求。基于这一特性，Engram构建三级缓存：

```
高速缓存(GPU HBM)     → 最常用嵌入向量（临时文件柜）
中速缓存(Host DRAM)   → 次常用嵌入向量（核心档案柜）
大容量存储(NVMe SSD)  →稀有的长尾模式（档案室）
```

这种分层存储既保证了高频资料的访问速度，又通过低成本存储实现了海量记忆的容量扩展。

## 五、U型缩放定律：MoE与Engram的最优配比

研究者定义了**分配比率ρ**：分配给MoE专家容量的参数比例（ρ=1代表纯MoE）。

通过实验发现了**U型缩放定律**：

```
纯MoE并非最优 → 存在最优混合点 → 20%-25%预算给Engram效果最佳
```

### 性能对比（Engram-27B vs MoE-27B）

在**等参数、等计算量**的严格条件下，Engram-27B全面超越MoE-27B：

| 任务类型 | Benchmark | 提升幅度 |
|---------|-----------|---------|
| 知识密集 | MMLU | +3.4 |
| 知识密集 | CMMLU | +4.0 |
| 通用推理 | BBH | +5.0 |
| 通用推理 | ARC-Challenge | +3.7 |
| 代码生成 | HumanEval | +3.0 |
| 数学推理 | MATH | +2.4 |

> **关键发现**：Engram的优势不仅体现在知识密集型任务，在需要复杂推理的任务上提升**更显著**。这证明Engram不仅是补充知识，更是提升了计算效率。

## 六、本质：增加模型的"有效深度"

为什么Engram能带来如此显著的性能提升？通过LogitLens和CKA等可解释性工具分析，核心答案浮出水面：

### 6.1 Engram等价于增加有效深度

传统模型的前几层大多在处理识别实体、组合短语这类低级任务。Engram把这些任务"外包"出去，让模型前几层直接开始更高层次的抽象推理。

**实验验证**：
- **LogitLens分析**：Engram模型在早期层级就表现出比MoE基线更低的KL散度，说明模型"更快地收敛到正确结果"
- **CKA分析**：Engram-27B的第5层表征与MoE基线的第12层最为接近——用更少物理层数实现了更深语义理解

### 6.2 释放注意力机制

Engram把局部依赖的处理外包后，注意力机制能专注于捕捉全局上下文的长距离依赖：

| 能力 | MoE-27B | Engram-27B |
|------|---------|-----------|
| 多查询NIAH | 84.2% | **97.0%** |

这对长文档处理场景（如法律文书分析、学术文献解读）价值巨大。

### 6.3 门控机制精准识别静态模式

可视化分析显示，处理以下内容时门控激活值显著升高：

- 英文：Alexander the Great、Princess of Wales
- 中文：四大发明、张仲景

直接证实Engram能精准识别并处理固化语言模式。

## 七、与RAG的本质区别

Engram与RAG（检索增强生成）常被拿来对比，但技术路径有本质区别：

| 维度 | Engram | RAG |
|------|--------|-----|
| **知识形态** | 嵌入表集成在模型层级 | 外部知识库 |
| **查找机制** | N-gram + O(1)哈希查找 | 向量相似度检索 |
| **延迟特性** | 确定性寻址，几乎无额外延迟 | 检索有延迟 |
| **知识更新** | 需重新训练 | 可实时更新 |
| **适用场景** | 静态、模式化事实 | 动态、长尾知识 |

**两者互补**：Engram优化静态知识的内在存储效率，RAG解决动态知识的实时调取。

## 八、对DeepSeek下一代模型的战略价值

业内普遍预测Engram将成为DeepSeek V4/R2的核心基础技术。结合mHC框架，下一代模型的架构轮廓已然清晰：

```
动态计算(mHC) + 静态检索(Engram) = 协同模式
```

### 性能提升路径

- mHC提升专家间通信效率
- Engram解决计算资源竞争
- 按U型定律，20%-25%参数分配给Engram

### 成本优化路径

- 训练中：避免静态知识冗余计算
- 推理中：O(1)检索比神经网络更省资源
- 千亿级记忆表可存于CPU，节省GPU显存

> **业内预测**：基于此架构的模型API调用成本或下降30%-50%

## 九、总结：条件记忆的范式意义

Engram的核心贡献在于重新定义了大模型的"任务分工"：

| 贡献维度 | 具体表现 |
|---------|---------|
| **性能提升** | 等参数等计算量下全面超越MoE，尤其复杂推理 |
| **效率优化** | 功能上等价于增加有效深度，释放注意力容量 |
| **系统突破** | 确定性寻址突破GPU内存墙，万亿级记忆表可低成本部署 |

### 核心启示

1. **稀疏性的新维度**：除了条件计算（MoE），条件记忆是另一条可行的稀疏化路径
2. **查算分离**：将静态知识存储与动态计算解耦，是突破效率瓶颈的关键
3. **架构收敛**：未来大模型将从"参数规模扩张"转向"精准分工优化"

Engram论文的开源（GitHub: deepseek-ai/Engram）不仅带来了新技术，更带来了新的研究思路——从神经网络的"一切皆计算"范式，向"计算+记忆"协同系统演进。
