### 11. MCP 企业级价值重估：为什么 CLI 替代 MCP 是误导

Motion 工程师 Charles Chen 发表长文，驳斥"MCP 已死、CLI 当立"的过度简化叙事。

### 6. LangChain：Harness Engineering 方法论

LangChain 发布两篇重量级文章，系统化阐述 **Harness Engineering**（工具架工程）方法论。

**核心公式**：`Agent = Model + Harness`

- 模型是引擎，Harness 是悬挂/传动——模型决定上限，Harness 决定能否触及上限
- Harness = System Prompts + Tools + Middleware（三大可调维度）
- LangChain 仅调优 Harness，在 Terminal Bench 2.0 上将得分从 52.8% 提升至 66.5%（模型不变）

Vivek Trivedy 的[《The Anatomy of an Agent Harness》](https://blog.langchain.com/the-anatomy-of-an-agent-harness/) 提供了最清晰的 Harness 定义框架；[《Improving Deep Agents with Harness Engineering》](https://blog.langchain.com/improving-deep-agents-with-harness-engineering/) 则用数据证明了工程化调优的 ROI。

> 本仓库已新增 [Agent Harness Engineering 深度解读](../articles/engineering/agent-harness-engineering.md)，内化了两篇文章的核心洞见。

### 13. 🔥 Meta 内部 AI Agent 触发 Sev 1 安全事件

### 12. Computer-Use：让 Agent 操作真实桌面界面的能力拐点

本周技术社区热议的一个关键趋势是 **Computer-Use**（计算机使用）能力——模型直接操控桌面 UI、点击软件、填写表单，而非通过专用 API。

**核心意义**：
- 大量企业软件没有干净的 API（Pre-API 时代的遗留系统）
- 现在 Agent 可以通过人类使用的相同界面操作任何软件
- 多应用工作流可以端到端自动运行，无需人工介入
- 最新前沿模型在桌面导航性能上已超越人类基准

这一能力对 Agent 开发者意味着：**无需为每个系统构建定制集成，只要软件能跑在屏幕上，Agent 就能操作它**。

> 来源：[dev.to — The LLM and AI Agent Releases That Actually Matter This Week, March 2026](https://dev.to/aibughunter/the-llm-and-ai-agent-releases-that-actually-matter-this-week-march-2026-5d7i)
